Déploiement vers Heroku¶

Heroku est un service d’hébergement cloud de type PaaS (Plateform as a Service). Dans ce chapitre nous allons créer une infrastructure en ligne et déployer un serveur Java à partir de heroku-cli (l’outil Heroku en ligne de commandes).

Créer un compte Heroku

Créez un compte sur le site de Heroku :

https://signup.heroku.com/

Installation de heroku-CLI¶

Avant d’installer heroku-cli, vous devez installer le client Git :

https://git-scm.com/downloads

Vous devez ensuite installer le client Heroku :

https://devcenter.heroku.com/articles/heroku-cli

Vous pouvez ensuite vous connecter à votre compte en ligne de commandes :

$ heroku login

Création d’une application Heroku¶

Pour créer une application, tapez simplement :

$ heroku create [NOM APP]

Avertissement

Si vous créez une application nommée monapp :

$ heroku create monapp

Elle sera disponible à l’adresse :

https://monapp.herokuapp.com

Il faut donc choisir un nom d’application qui ne soit pas déjà utilisé.

Il est possible de supprimer définitivement une application :

$ heroku apps:destroy -a [NOM APP]

Déploiement d’une application¶

Une application Heroku est déployée via Git. Chaque application est associée à un dépôt Git sur la plate-forme de Heroku.

Pour connaître la liste de vos applications :

$ heroku apps

Pour cloner le dépôt d’une de vos applications :

$ heroku git:clone -a [NOM APP]

Chaque fois que vous réalisez un push sur le dépôt de Heroku dans la branche master, votre application est automatiquement redéployée. Pour un projet Java, Heroku va lancer la commande Maven suivante sur son serveur :

mvn clean package

Il va ensuite lire un fichier Procfile qui doit être présent à la racine du projet (à côté du fichier pom.xml) pour déterminer la commande à exécuter pour lancer le programme. Nous verrons dans le dernier exercice ce qu’il faut écrire dans le fichier Procfile.

Note

En fait, Heroku ne fournit pas d’environnement Java EE proprement dit. Il permet d’exécuter des programmes Java en ligne de commandes (la commande étant décrite dans le fichier Procfile). À charge de l’application de fournir le serveur complet pour s’exécuter.

Associer à un dépôt Git existant¶

La plupart du temps, le code source de l’application est déjà hébergé sur un dépôt Git distant. Il est possible d’associer en local un même projet à plusieurs dépôts distants. Nous avons ensuite le choix du dépôt distant vers lequel nous voulons envoyer les modifications.

Si vous créez votre application dans le répertoire du projet que vous avez cloné :

$ cd workspace/mon_projet
$ heroku create mon_projet

L’appel à la commande create va automatiquement ajouter un dépôt distant (remote) appelé heroku. Vous pourrez ainsi déployer votre nouveau code avec la commande :

$ git push heroku master

Si vous avez déjà créé votre application sur Heroku, alors il suffit de vous déplacer dans le répertoire dans lequel le projet a été cloné et de taper la commande suivante :

$ cd workspace/mon_projet
$ heroku git:remote -a mon_projet

Comme précédemment, la commande git:remote va automatiquement ajouter un dépôt distant (remote) appelé heroku. Vous pourrez ainsi déployer votre nouveau code avec la commande :

$ git push heroku master

Déployer une application sur Heroku

Créez une application sur Heroku à partir d’un projet Spring Boot. Le déploiement de l’application se fait simplement par la commande git push heroku master.
